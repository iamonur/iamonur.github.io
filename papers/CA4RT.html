A brief summary

  • Abstract
    • Uses cellular automata for PCG since it is computationally efficient. Thus, CA can run real-time.
  • Introduction
    • Some kind of games require real-time generation of levels (ie rogue-like ones.)
    • Since CA-PCG is inexhaustable, levels become less predictable.
    • If PCG is used in level generation, levels become parameterazible.
    • Also, PCG can be used as a guidline for human level-designers.
  • Background
    1. One common algorithm is Mid-point Displacement:
      • It's easy to implement and computationally efficient.
      • It is highly indirect
        • Only parameter it takes is a random seed.
        • It is not controllable, so it is not a good approach for PCG.
    2. Another one is Search-Based PCG(SBPCG):
      • Consumes lots of computational power.
      • Highly controllable.
    3. CA for PCG:
      • This is the first example of it in academy. From what I see, it is both controllable enough and computationally efficient.
  • CA-PCG Approach
    • Initial grid is called base.
    • 4 neighbors for basic CA, 8 for advanced. Also you can increase it to 24, for 2 unit distance; or 48 for 3 unit distance, etc.
    • This is how the generation works
      1. Initialize the level empty.
      2. Change r% of the level to rock(R=50 by default.)
      3. Iterate Cellular Automata n times.
        1. Calculate "Neighborhood Value" for every cell
        2. If NV is less than T (T=5 by default), make this cell a "floor"
        3. Else, makes this cell a rock.
      4. Draw a wall around rocks.
    • Given the same parameter set(random seed, r, n, T, and M(Moore Neighborhood Size)), the same level can be reconstructed.
  • Maps with different parameters are presented are presented in the paper.
