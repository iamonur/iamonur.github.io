A brief summary

  • Abstract
    • Analysis of how to fine-tune parameters of a game for optimal difficulty.
    • Works on a parameterized version of Flappy Bird.
    • An automated player with human motor skills plays for a Monte Carlo simulation. Score histograms are worked on.
  • Introduction
    • Traditional game design: Iterative manner. Design, play-test, get feedback, repeat.
    • Purpose of the work is to automate this cycle without human testers.
    • All parameters in game creates another dimension. This high-dimensional space is called game space. All points in game space are unique designs.
    • The reason of game choice is to have less parameters to tune.
    • Score probability distribution is a nice to have since it is both quantitative and easy to analyze.
    • Survival analysis provides a well-defined hazard rate.

  • Flappy Bird Game Space
    • Measured difficulty h(x) is probabilty of fail, given that score x is already reached. Called hazard rate.
    • Even a simple game like Flappy Bird has lots of parameters:
      1. World height
      2. Pipe gap location range: vertical distance between lower pipe in the obstacle before and upper pipe in next.
      3. Pipe gap
      4. Pipe seperation
      5. Pipe width
      6. Gravity
      7. Bird size
      8. Bird velocity
      9. Bird jump velocity
  • Player Model
    • Player Precision
      • This is how you deviate when you plan to press a button.
      • Standard deviation here is minimum 35.9 ms to 61.1 ms
      • When a button needs to be pressed at time T, they press it in t+€, where € is a random that is pulled from a normal distribution with 0 mean and the given standard deviation.
      • stdev=0 tests if level is playable, =35.9 is an expert player, =61.1 is a novice player
    • Actions Per Second and Reaction Time
      • A person should have 7.7 actions in a second at maximum.
      • When a change happens on the screen, it takes 288 ms to react, like a person.
  • Survival Analysis
    • f(x) is the probability distribution function. This is the probability of getting x as points in the game.
    • S(x) is surviving through point greater or equal to x.
    • Hazard function is h(x)=f(x)/S(x)
    • If h(x) = undefined, game is unwinnable
  • Method for Estimating Hazard Rates
    • Steps:
      1. Generate a game by parameters
      2. Simulate
      3. Analyze the histogram
